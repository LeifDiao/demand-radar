# 红线 —— 禁止的做法

这个 skill 的全部价值建立在「可信」上。下面任何一条被破坏，报告就从决策依据退化成自嗨材料。

## 数据红线

1. **禁止编造数据**。没有任何数字、引用、原话可以是你想象出来的。每个数字都要能追到一个 URL。
2. **无来源不下判断**。如果某个评分项没有证据，标「数据不足」，不许猜分、不许用常识补。
3. **不把「没查到」写成「不存在」**。检索不到证据 ≠ 需求不存在，如实写进「未能验证的部分」。
4. **不把意见证据当硬证据**。「我觉得会买」不能用来支撑「有付费意愿」的结论。
5. **引用要可验证**：原话 + 可点开的 URL + 日期。死链或二手转载要标注。

## 偏误红线

6. **不替用户自我说服**。默认假设是「这可能是伪需求」，由证据推翻它，而不是反过来找证据捧场。
7. **必须过红队**。任何「Go」裁决都要先经过证伪攻击，扛住了才算数。
8. **不掩盖矛盾证据**。自上而下和自下而上打架时，要高亮，不要挑对自己有利的那个写。
9. **不被 echo chamber 带偏**。HN/创业圈爱聊工具，不代表大众市场，需自上而下数据校准。

## 语气红线

10. **不用 AI 陈词**：「打造」「赋能」「全方位」「一站式」「无缝」「革命性」「颠覆」这类词一律不用。
11. **不过度承诺**。裁决要配置信度，低证据就给低置信度，别为了好看抬高结论。
12. **推测和事实分开写**。「数据显示 X」和「我推测 X」必须用不同措辞，读者要能分清。

## 自检（出报告前过一遍）

- [ ] 每个数字都有 URL 吗？
- [ ] 每个评分项要么有证据要么标了「数据不足」吗？
- [ ] Go 裁决过红队了吗？
- [ ] 有「未能验证的部分」这一节吗？
- [ ] 有没有把「会买」当成「买了」？
- [ ] 有没有 AI 陈词？
